História

O JavaScript foi inicialmente desenvolvido pelas empresas Sun Microsystems (depois adquirida pela Oracle) em conjunto com a Netscape Communications, que à época eram, respectivamente, a principal desenvolvedora de soluções na linguagem Java e a líder no mercado de navegadores web.

O JavaScript foi originalmente concebido como uma interface de scripting entre páginas web carregadas no navegador (o Netscape, à época) e a aplicação no servidor. A primeira versão lançada em 1995 tornou o JavaScript uma linguagem central do desenvolvimento web que mais tarde passaria a ser usada de forma nativa em servidores e dispositivos móveis.

À época de suas primeiras formulações, a Netscape enviou a especificação da linguagem para ser padronizada na ECMA, um órgão europeu formado por fabricantes de computadores. Devido ao nome "JavaScript" ser uma marca registrada da Sun, a versão padronizada da linguagem recebeu o nome de "ECMAScript".

A especificação está em sua versão 13, lançada em 22 de junho de 2022.

A versões da especificação ECMA até a versão 4 foram as mais rudimentares. Para manter a compatibilidade com códigos antigos, não podem ser feitas alterações que eliminem completamente alguns problemas iniciais, mas na versão ES5 e nas posteriores, passou a existir um modo estrito onde alguns destes erros são sanados.

A versão ES6 da especificação é uma das mais importantes na história da linguagem pois trouxe muitas funcionalidades do paradigma orientado a objetos, possibilitando o uso do JavaScript para implementar sistemas completos com estruturas de dados mais complexas.

A partir da versão ES6, o uso de novas funcionalidades automaticamente ativa o modo estrito, tornando as falhas das versões pré-ES5 como código legado. Ao usar, por exemplo, a palavra reservada class ou ao criar um módulo na versão ES6, todo código da classe ou módulo é automaticamente considerado estrito, e as inconsistências das versões iniciais ficam indisponíveis.
