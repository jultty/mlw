Nível de abstração

É uma linguagem de alto nível, com múltiplas camadas de abstração. Ele depende de um interpretador, geralmente contido em um navegador, para que suas instruções cheguem ao sistema operacional. Apesar de facilitar a execução e manutenção, isso também significa que a linguagem é menos performática do que suas alternativas de baixo nível para tarefas intensas que requerem um uso mais eficiente de memória.

Execução

Ao contrário de uma linguagem compilada, o JavaScript não produz arquivos binários quando executado, sendo portanto considerado uma linguagem interpretada. A primeira implementação de um motor para processar código JavaScript, chamada SpiderMonkey, foi feita por Brendan Eich em 1995, usando um interpretador simples.

A maior mudança neste ponto foi com o desenvolvimento do V8, criado pela Google para ser o motor JavaScript do Chrome. Desde então é possível utilizar um método de compilação em tempo de execução chamado just-in-time (JIT) ou tradução dinâmica para obter ganhos de performance na execução do código.

Posteriormente, o método foi uma característica importante do motor Nitro para o navegador Safari, que por sua vez teve parte do código adaptado na atualização do SpiderMonkey, utilizado até hoje no Mozilla Firefox.

Estrutura de dados

JavaScript é uma linguagem com tipagem dinâmica, o que significa que suas variáveis podem ser declaradas sem que o tipo seja explicitamente definido. O interpretador é capaz de deduzir a partir do valor atribuído qual é o tipo de dado para cada variável.

Há suporte para diferentes estruturas de dados, tanto na forma de tipos primitivos quanto de variáveis compostas como vetores e matrizes. Por padrão, as variáveis compostas são heterogêneas, podendo conter dados de diferentes tipos.

A linguagem também possui um comportamento denominado hoisting, que permite que variáveis e funções sejam utilizadas anteriormente à sua declaração.

Paradigma

Considerada com uma linguagem multiparadigma, o JavaScript não nasceu pensado para uma metodologia de desenvolvimento específica. A linguagem pode ser utilizada no paradigma funcional, estruturado, orientado a objetos, entre outros.

Em 2015, a versão 6 da especificação ECMAScript trouxe muitas funcionalidades do paradigma orientado a objetos, como classes e funções nativas para que as propriedades de uma classe possam ser herdadas por seus objetos.
