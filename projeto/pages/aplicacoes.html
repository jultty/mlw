Aplicações

O JavaScript pode ter tantas aplicações diferentes que não seria possível cobri-las no escopo deste projeto. Portanto, serão abordadas apenas as duas grandes distinções de ambientes onde ele pode ser implementado: como uma linguagem para web, executada dentro de um navegador, e como código no lado do servidor, através de tecnologias como o Node.js.

JavaScript front-end

O ambiente mais comum para o JavaScript é o navegador web. Ele permite que o código receba entradas do mouse e teclado e também através de requisições HTTP. Como saídas, a linguagem pode ser usada para manipular elementos HTML e suas propriedades CSS.

Para ser executado, o código JavaScript precisa estar contido ou linkado em uma página HTML. Em um exemplo real, isso significa que quando acessamos uma página na internet estamos solicitando que cópias de todo seu código HTML, CSS e JavaScript sejam transmitidas para nosso navegador.

Quando o navegador recebe os pacotes solicitados, ele começa um processo de análise sintática denominado parsing. Isso significa que o navegador irá verificar todos os elementos contidos nas tags HTML e compará-los com a sintaxe da linguagem.

No caso do JavaScript, é ao encontrar as tags <script> que o navegador irá analisar esse tipo de código.

Ao finalizar a análise sintática, o navegador cria modelos de todo o código HTML e CSS na memória, e então compila e executa o código JavaScript.

JavaScript back-end

Apesar de usar a mesma linguagem, essa forma de trabalhar com o JavaScript é diferente da sua implementação front-end por não depender de um navegador. Nesse caso, é possível executar diretamente o código JavaScript a partir de um arquivo .js e obter o retorno no console do próprio sistema operacional.

Uma plataforma nativa de JavaScript significa que a linguagem pode agora executar processos e subprocessos no sistema operacional e realizar operações de entrada e saída não somente em um navegador web mas a partir das suas interfaces de rede, armazenamento e mídia.

Se instalado em um servidor, o Node.js permite ainda que o JavaScript seja usado para enviar e receber requisições HTTP, podendo portanto ser usado para criar aplicações web que interligam diferentes sistemas (APIs).

Por ser capaz de fazer conexões com bancos de dados, o Node.js também pode substituir código PHP para uma integração mais uniforme com sistemas que já utilizam JavaScript para outras funcionalidades. É possível ainda implementar servidores usando Node.js, reduzindo ainda mais a complexidade de diferentes linguagens e módulos com uma única tecnologia.
